## 🚀 Getting Started

### Step 1: Run Security Setup
```bash
npm run setup
```
This will:
- Generate secure JWT and CSRF secrets
- Create `.env` file from template
- Display next steps

### Step 2: Install Dependencies
```bash
npm install
```
New dependencies:
- `dotenv` - Environment variable management
- `@fastify/helmet` - Security headers

### Step 3: Configure Environment
Edit `.env` file and update:
```env
DATABASE_URL=postgres://user:password@localhost:5432/dbname
FRONTEND_URL=http://localhost:3000
NODE_ENV=development
```

### Step 4: Start Server
```bash
# Development mode
npm run dev

# Production mode
NODE_ENV=production npm start
```

## 🔐 Security Features Overview

### 1. Environment Variables
All secrets are now in `.env`:
- JWT_SECRET (auto-generated)
- CSRF_SECRET (auto-generated)
- DATABASE_URL
- SSL certificate paths

### 2. Input Validation
Every endpoint validates:
- Email format
- URL format
- String lengths
- Required fields
- Data types

### 3. CSRF Protection
Get token before making requests:
```javascript
// 1. Get CSRF token
const { csrfToken } = await fetch('/api/csrf-token', {
  headers: { 'Authorization': `Bearer ${token}` }
}).then(r => r.json());

// 2. Include in POST/PUT/DELETE requests
fetch('/api/scans', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'X-CSRF-Token': csrfToken,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(data)
});
```

### 4. Rate Limiting
Automatic per-user limits:
- **Free**: 5 scans/day, 50 API requests/hour
- **Basic**: 20 scans/day, 200 API requests/hour
- **Premium**: 100 scans/day, 1000 API requests/hour
- **Enterprise**: Unlimited scans, 5000 API requests/hour

### 5. File Upload Security
Only allows:
- PDF, JSON, TXT, CSV, XLSX
- Max 10MB per file
- Validates MIME types
- Scans for malicious content

### 6. SQL Injection Prevention
All queries use parameterized statements:
```javascript
// ✅ Safe
await client.query('SELECT * FROM users WHERE id = $1', [userId]);

// ❌ Never do this
await client.query(`SELECT * FROM users WHERE id = ${userId}`);
```

### 7. Error Handling
- Production: Generic error messages
- Development: Detailed error info
- No stack traces exposed to clients

### 8. Security Headers
Automatic headers via Helmet:
- Content-Security-Policy
- X-Frame-Options
- X-Content-Type-Options
- Strict-Transport-Security

## 🔧 Production Setup

### Enable HTTPS
1. Get SSL certificates (Let's Encrypt recommended)
2. Update `.env`:
   ```env
   ENABLE_HTTPS=true
   SSL_KEY_PATH=/path/to/key.pem
   SSL_CERT_PATH=/path/to/cert.pem
   ```

### Generate SSL Certificate (Development)
```bash
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes
```

### Production Checklist

## 🧪 Testing Security

### Test Input Validation
```bash
# Should fail with invalid email
curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"invalid","password":"test","role":"developer"}'
```

### Test Rate Limiting
```bash
# Make 60 requests quickly - should get rate limited
for i in {1..60}; do
  curl http://localhost:5000/api/scans \
    -H "Authorization: Bearer YOUR_TOKEN"
done
```

### Test CSRF Protection
```bash
# Should fail without CSRF token
curl -X POST http://localhost:5000/api/scans \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"target":"http://example.com"}'
```
